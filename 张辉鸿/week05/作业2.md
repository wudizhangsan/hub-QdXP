## 作业2：基于 BERT 的文本编码与相似度计算技术方案



### 1. 技术架构概述

方案分为**离线向量化（Offline Indexing）**与**在线检索（Online Retrieval）**两个阶段。核心逻辑是将变长的文本序列映射为定长的稠密向量，使得语义相似的文本在向量空间中距离更近。

### 2. BERT 文本编码流程 (Encoding)

要使用 BERT 进行编码，需经历以下步骤：

1. **输入预处理 (Tokenization)**：
   - 使用 BERT 自带的 Tokenizer 对中文文本进行分词（字粒度）。
   - 在文本序列首部添加特殊标记 `[CLS]`（用于分类/聚合），尾部添加 `[SEP]`（用于分隔）。
   - 例如：“忘记密码” $\rightarrow$ `[CLS] 忘 记 密 码 [SEP]`。
2. **模型前向传播**：
   - 将 Token ID 序列输入 BERT Base 模型（12层 Transformer）。
   - 模型结合上下文注意力机制，输出每个 Token 对应的 768 维（Hidden Size）向量。
3. **向量池化 (Pooling)**：
   - 为了得到整句话的语义表示，通常提取 **`[CLS]` 标记对应的输出向量**，或者计算所有 Token 输出向量的**平均值 (Mean Pooling)**。此向量即为该文本的 Embedding。

### 3. 相似度计算 (Similarity Calculation)

获得向量后，通过数学距离判断相似性：

- 假设用户 Query 的向量为 $V_q$，知识库中某问题的向量为 $V_k$。

- 采用 **余弦相似度 (Cosine Similarity)** 公式：

  $$\text{Score} = \frac{V_q \cdot V_k}{\|V_q\| \times \|V_k\|}$$

- **判定逻辑**：

  - 计算 Query 与库中所有问题的相似度得分（范围 -1 到 1）。
  - 对得分进行降序排列。
  - 若最高分 $>\text{阈值}$（例如 0.85），则返回该问题对应的答案；否则提示“未找到相关问题”。
