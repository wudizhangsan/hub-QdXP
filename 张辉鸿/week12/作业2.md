### 1. 前后端分离

把项目分成**前端**和**后端**两个独立部分：

- **前端**：只做页面展示、用户输入、界面交互，不处理数据逻辑；

- **后端**：只做业务计算、数据库操作、调用大模型，通过 API 接口给前端返回数据。

  两者分开开发、分开运行、分开部署。

------

### 2. 历史对话如何存储 + 如何传给大模型

#### （1）历史对话存储

- 用**字典 / 列表**（内存）或**数据库**（持久化）保存；

- 按 **用户 ID** 分组，每条记录固定格式：`角色(用户/助手)` + `对话内容`；

- 示例结构：

  ```
  用户1: [
    {"role": "user", "content": "帮我分析茅台股价"},
    {"role": "assistant", "content": "当前股价XXX..."}
  ]
  ```

#### （2）作为大模型下一次输入

1. 取出该用户**全部历史对话**；
2. 在历史后面**追加本次新问题**；
3. 把拼接好的完整列表**直接传给大模型**；
4. 模型返回结果后，再把这条新回复**存入历史**
