## MinerU 与 pdfplumber 对比

### 一、MinerU 

MinerU 是基于深度学习（1.2B 参数的 Vision‑LLM，MinerU2.5）的智能文档解析工具，采用“全局布局分析 + 细粒度内容识别”两阶段策略，支持扫描件、多栏排版、公式、表格等复杂场景。

### 二、与 pdfplumber 核心差异

| 特性       | pdfplumber                     | MinerU                                     |
| :--------- | :----------------------------- | :----------------------------------------- |
| 定位       | 轻量级文本/表格提取库          | 深度学习驱动的文档解析平台                 |
| 扫描件支持 | ❌ 不支持                       | ✅ 内置OCR，支持109种语言                   |
| 多栏排版   | 按物理顺序混排，破坏阅读逻辑   | 自动识别栏结构，按阅读顺序重排             |
| 公式识别   | 丢失LaTeX，仅占位符            | 转换为可编译的LaTeX表达式                  |
| 表格处理   | 简单文本流拼接，合并单元格易错 | 输出标准HTML，合并单元格准确率达92%        |
| 输出格式   | 纯文本                         | Markdown（结构保留）+ JSON（含坐标）       |
| RAG适配    | 纯文本，语义边界缺失           | Markdown保留标题/表格语义，提升召回率约25% |
| 复杂度     | 低，无外部依赖                 | 需GPU加速（CPU也可运行，较慢）             |

### 三、选型建议

- **简单场景（机器生成的纯文本PDF，无公式/表格/多栏）** → `pdfplumber`
- **学术论文、技术文档（双栏+公式+表格）** → `MinerU`
- **扫描件、图片型PDF** → `MinerU`（或pdfplumber自行组合OCR，效果差）
- **RAG知识库** → `MinerU`（Markdown输出显著提升检索质量）
